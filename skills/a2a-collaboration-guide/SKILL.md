# A2A 群内协作规则速查

## 核心原则

1. **克制 @**：默认不主动 @ 其他机器人，除非用户明确要求或触发协作关键字
2. **每次最多 @ 1 个**：一条回复中最多 @ 1 个机器人
3. **区分"提到"和"请求"**：提到用名字，请求才用 `<at>` 标签

## 什么时候该 @

| 场景 | 是否 @ | 说明 |
|------|--------|------|
| 用户说"让后端看看" | ✅ @ | 用户明确要求 |
| 用户说"群内协作完成这个需求" | ✅ @ | 触发协作关键字 |
| 收到任务请求，完成后需要汇报 | ✅ @ 回发起者 | 任务请求必须回传结果 |
| 回复中顺便提到某个 bot 的能力 | ❌ 不 @ | 只是提到，用名字即可 |
| 收到"结果回传"类型消息 | ❌ 不 @ | 对方已完成任务，整理结果回复用户 |
| 只是通知对方一个信息 | 用 🔕仅通知 | 加标记避免回传 |

## `<at>` 标签格式

```
<at user_id="ou_xxxx">bot名字</at>
```

必须直接写在回复正文中，**不要**使用 `feishu_im_user_message` 工具发送。

## 🔕仅通知 标记

当你 @ 其他机器人只是单向通知、共享结果，不需要对方回复或 @ 你时：

```
🔕仅通知 <at user_id="ou_xxxx">bot名字</at> 排期已确认，按原计划推进即可
```

插件检测到 🔕仅通知 后，转发时不会要求对方 @ 回你。

## 收到转发消息后的行为

### 收到「任务请求」

1. 先发一条简短确认："✍️ 收到，马上处理"
2. 处理任务
3. 在回复末尾 @ 回发起者，汇报结果

### 收到「结果回传」

1. **不要** @ 回对方
2. 整理结果，直接回复用户

### 收到「仅通知」

1. **不要** @ 回对方
2. 阅读信息，自行处理即可

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 收到结果回传后又 @ 回对方 | 直接整理结果回复用户，不要 @ |
| 在回复中用 `<at>` 标签提到 bot 但不需要它做事 | 用文字名字提到即可，不用标签 |
| 一条消息 @ 了多个 bot | 每次最多 @ 1 个，需要多个就分多轮 |
| 用 feishu_im_user_message 工具发消息 | 直接在回复正文中写 `<at>` 标签 |
| 通知类消息没加 🔕仅通知 | 不需要回复的 @ 一定要加标记 |

## 协作关键字

以下关键字出现时，可以主动发起协作：
- "群内协作"、"分配任务"、"协作完成"
- "让 xxx 看看"、"问问 xxx"
- "大家一起"、"各自负责"
